# Support

Thanks for using Huddle! Here's how to get help — and what to expect.

## No SLA — provided "AS IS"

Huddle is free and open source software, provided **"AS IS", without warranty of
any kind**, as described in the [GNU GPL v3](LICENSE) (see sections 15–17).

- There is **no service-level agreement (SLA)**.
- Support is provided by volunteers on a **best-effort basis**.
- Issues and pull requests are **welcome**, but there is **no guarantee** of a
  response time, a fix, or that any given feature will be implemented.
- Use in production is at your own risk.

## Where to get help

| I want to… | Go to… |
|------------|--------|
| Report a bug | Open an issue with the **Bug report** template. |
| Request a feature | Open an issue with the **Feature request** template. |
| Ask a usage question | Open a **Question** issue (or use Discussions if enabled). |
| Report a security issue | **Do not** open a public issue — see [`SECURITY.md`](SECURITY.md). |
| Understand how Huddle works | Read the [README](README.md). |
| Contribute | Read [`CONTRIBUTING.md`](CONTRIBUTING.md). |

## Before opening an issue

1. Check the [README](README.md) and its Troubleshooting / FAQ sections.
2. Search existing [issues](https://github.com/infosupport/huddle/issues)
   (open **and** closed).
3. Make sure you're on the latest released version.

Well-scoped, reproducible reports are the most likely to get attention.
